## Установка и запуск

### Требования

- **Node.js** 20.19.0+ (LTS)
- **Python** 3.10+ (для сборочных скриптов VS Code)
- **Windows 10/11** (основная платформа), Linux и macOS поддерживаются

### Запуск из исходников

```bash
cd vscode
npm install
npm run compile
./scripts/code.bat       # Windows
./scripts/code.sh        # macOS/Linux
```

### Сборка релиза

```bash
cd vscode
npm run gulp vscode-win32-x64   # Сборка для Windows x64 → .build/
```

## Настройка моделей

Все модели настраиваются в файле `~/.xynapse/config.yaml`. Шаблон конфигурации находится в расширении: `vscode/extensions/xynapse-assistant/xynapse-config.yaml`.

### Пример подключения модели

```yaml
models:
  - name: "DeepSeek V3.2"
    provider: "yandex_gpt"
    model: "deepseek-v3.2/latest"
    apiKey: "$YANDEX_API_KEY"
    roles:
      - chat
      - edit
    defaultCompletionOptions:
      temperature: 0.3
      maxTokens: 8192
    requestOptions:
      extraBodyProperties:
        folderId: "$YANDEX_FOLDER_ID"
```

### Поддерживаемые провайдеры

| Провайдер | Описание |
|-----------|----------|
| `yandex_gpt` | Yandex Cloud Foundation Models (YandexGPT, DeepSeek, Qwen, Gemma и др.) |
| `openai` | OpenAI-совместимые API (GPT-4, GPT-4o и др.) |
| `ollama` | Локальные модели через Ollama |
| `lmstudio` | Локальные модели через LM Studio |
| `anthropic` | Anthropic Claude |
| `openrouter` | Маршрутизация через OpenRouter |

### API-ключи

Ключи можно задать тремя способами:

1. **Переменные окружения** (рекомендуется):
   ```
   YANDEX_API_KEY=ваш-ключ
   YANDEX_FOLDER_ID=ваш-folder-id
   ```

2. **Файл секретов** — `secrets.local.json` в папке расширения:
   ```json
   {
     "apiKey": "ваш-ключ",
     "folderId": "ваш-folder-id"
   }
   ```

3. **Напрямую в config.yaml** (не рекомендуется для общих конфигов):
   ```yaml
   apiKey: "ваш-ключ-прямо-здесь"
   ```

## Роли моделей

Каждая модель может выполнять одну или несколько ролей:

| Роль | Описание | Пример использования |
|------|----------|----------------------|
| `chat` | Основная роль для чата, плана и агента | Диалог с ассистентом (Ctrl+L) |
| `edit` | Редактирование выделенного кода | Встроенное редактирование (Ctrl+I) |
| `autocomplete` | Подсказки кода при наборе | Tab-автодополнение |
| `apply` | Применение сгенерированных блоков кода | Вставка кода в файлы |
| `summarize` | Суммаризация длинных контекстов | Сжатие истории чата |
| `embed` | Создание эмбеддингов для поиска | @codebase и @docs |
| `rerank` | Переранжирование результатов поиска | Улучшение релевантности @codebase |

### Назначение ролей

```yaml
models:
  - name: "YandexGPT Pro"
    provider: "yandex_gpt"
    model: "yandexgpt/latest"
    roles:
      - chat       # Используется для чата
      - edit       # И для редактирования кода
```

Если роль не назначена ни одной модели, используется модель чата по умолчанию.

## Конфигурация config.yaml

Полная структура файла `~/.xynapse/config.yaml`:

```yaml
name: "My Config"
version: "1.0.0"
schema: "v1"

# Модели
models:
  - name: "Имя модели"
    provider: "провайдер"
    model: "идентификатор-модели"
    apiKey: "$ENV_VAR"
    roles: [chat, edit]
    defaultCompletionOptions:
      temperature: 0.3
      maxTokens: 8192

# Контекстные провайдеры
context:
  - provider: "code"
  - provider: "docs"
  - provider: "diff"
  - provider: "terminal"
  - provider: "problems"
  - provider: "folder"
  - provider: "codebase"

# Слеш-команды
slashCommands:
  - name: "edit"
    description: "Редактировать выделенный код"
  - name: "explain"
    description: "Объяснить выделенный код"

# Экспериментальные настройки
experimental:
  contextMenuPrompts:
    comment: "Добавь комментарии к коду"
    docstring: "Напиши docstring"
    fix: "Исправь ошибки"
    optimize: "Оптимизируй код"
    fixGrammar: "Исправь грамматику"

# Правила для ассистента
rules:
  - "Всегда отвечай на русском языке"
  - "Используй понятные комментарии в коде"
```

## Контекстные провайдеры

Контекстные провайдеры добавляют информацию в запрос к модели. Используйте `@` в чате для их вызова.

| Провайдер | Команда | Описание |
|-----------|---------|----------|
| `code` | `@code` | Поиск по кодовой базе |
| `docs` | `@docs` | Поиск по подключённой документации |
| `diff` | `@diff` | Текущие изменения (git diff) |
| `terminal` | `@terminal` | Содержимое терминала |
| `problems` | `@problems` | Ошибки и предупреждения из панели проблем |
| `folder` | `@folder` | Содержимое выбранной папки |
| `codebase` | `@codebase` | Семантический поиск по всей кодовой базе |
| `file` | `@file` | Содержимое конкретного файла |
| `url` | `@url` | Содержимое веб-страницы |

## Режимы работы

Xynapse Assistant поддерживает несколько режимов:

| Режим | Описание |
|-------|----------|
| **Чат** | Обычный диалог с ассистентом. Подходит для вопросов, объяснений, генерации кода |
| **Агент** | Автономное выполнение задач. Ассистент может читать файлы, выполнять команды, редактировать код |
| **План** | Создание плана перед выполнением. Ассистент анализирует задачу и предлагает пошаговый план |
| **Фоновый** | Выполнение задач в фоновом режиме без блокировки интерфейса |

Переключение между режимами — через селектор в нижней части окна чата.

## Индексация кодовой базы

Индексация позволяет ассистенту искать по всей кодовой базе через `@codebase`.

### Включение

В настройках ассистента → **Индексация** → включите переключатель.

### Требования

- Настроенная модель эмбеддингов (роль `embed`)
- Встроенная модель `Transformers.js` работает из коробки

### Переиндексация

Если кодовая база изменилась, нажмите «Нажмите для переиндексации» в секции индексации.

### Примечание

Индексация может потреблять значительные ресурсы на больших проектах. На маломощных машинах рекомендуется отключить её и использовать `@file` или `@folder` вместо `@codebase`.

## Настройки ассистента

Доступны через иконку шестерёнки в панели ассистента.

### Вкладки настроек

| Вкладка | Описание |
|---------|----------|
| **Модели** | Управление моделями и их ролями |
| **Инструменты** | MCP-серверы и встроенные инструменты |
| **Правила** | Правила поведения ассистента |
| **Настройки** | Параметры чата, автодополнения и UI |
| **Конфигурации** | Профили config.yaml |
| **Индексация** | Индексация кодовой базы и документации |
| **Справка** | Горячие клавиши, ресурсы, диагностика |

### Основные настройки

- **Показывать полосу прокрутки чата** — отображение скроллбара в окне чата
- **Автодополнение** — вкл/выкл подсказок кода при наборе
- **Отключение файлов** — glob-шаблоны для отключения автодополнения в определённых файлах

## Council — мультиагентное обсуждение

Команда `/council` запускает мультиагентную сессию для планирования сложных задач.

### Алгоритм: 3 фазы

1. **Независимый анализ** — каждый агент анализирует задачу отдельно
2. **Перекрёстная критика** (0–2 раунда) — агенты видят ответы друг друга и критикуют
3. **Синтез плана** — планировщик собирает итоговый план, разрешая конфликты

### Уровни сложности

| Уровень | Фаза 1 | Фаза 2 (раундов критики) | Всего вызовов LLM |
|---------|--------|--------------------------|---------------------|
| easy | N агентов | 0 | N + 1 |
| medium | N агентов | 1 | 2N + 1 |
| hard | N агентов | 2 | 3N + 1 |

### Выходные файлы

- `council-plan.md` — итоговый план (открывается автоматически)
- `council-discussion.md` — полный лог обсуждения

## Устранение неполадок

### Модели не отображаются в GUI

1. Проверьте `~/.xynapse/config.yaml` — файл должен быть валидным YAML
2. Убедитесь, что API-ключи заданы корректно
3. Перезагрузите окно: `Ctrl+Shift+P` → `Reload Window`

### Ассистент не отвечает

1. Проверьте подключение к интернету (для облачных моделей)
2. Откройте Extension Host Output (`Ctrl+Shift+P` → `Show Logs`) и ищите ошибки с префиксом `[Xynapse]`
3. Попробуйте другую модель

### Автодополнение не работает

1. Убедитесь, что назначена модель с ролью `autocomplete`
2. Проверьте, что автодополнение включено: `Ctrl+K, Ctrl+A`
3. Некоторые типы файлов могут быть исключены через настройку `disableInFiles`

### Индексация зависает

1. На больших проектах (>10000 файлов) индексация может занять время
2. Попробуйте уменьшить область через `.gitignore`
3. Если индексация не завершается — отключите и используйте `@file`/`@folder`

### Логи для диагностики

- **Backend**: Extension Host Output → ищите `[Xynapse]`
- **Frontend**: DevTools (`F12`) → Console → ищите `[Xynapse GUI]` и `[Xynapse ModelSelect]`
